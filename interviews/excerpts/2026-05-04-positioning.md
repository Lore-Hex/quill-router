# Positioning Excerpts

Source: `interviews/transcripts/2026-05-04-positioning.md`

## Core Sentence

TrustedRouter is one API, all the LLMs, provably private.

## Router Value

TrustedRouter lets you route to any LLM very easily. Switch between all your LLMs, have higher uptime, have backups, and try out much cheaper models, saving you money without needing to trust TrustedRouter servers as an intermediary any more than you need to.

## Open Source

Most importantly it's all open source software. Every part of the backend infrastructure, configuration, bring up, and UI is entirely open source.

## OpenRouter-Compatible But Verifiable

The wedge: we make it super easy to one-line replace your call to any direct LLM and immediately have it be a router that can handle all of it transparently.

You have no idea really what it's doing in terms of your prompt and how safely it's carrying it out. You can't really know whereas for our code you know end to end what's happening with open source and you know that it's running.

## First Users

Many engineers are using OpenRouter but only for low-security non-sensitive data because they know it's not so safe to put sensitive data in this third-party router.

## Opaque Routers

We should really demand that all routers be totally open source. That's really the only way to be sure that it is running things that are safe and secure.

Many eyes catch all bugs and by having the whole community be able to verify all pieces of the infrastructure then we can know it's secure.

## Attested Router

On top of that we have provable security by having secure enclave attested security around all of the code that's running. It's a very minimal amount of code in the attested router.

## What Not To Claim

We can never claim that we're perfectly secure but we can claim that we're perfectly transparent and being open source and open about everything that we're doing.

## Switch This Week

We're meeting a lot of people who are using OpenRouter or something like it but only for a fraction of their use cases. They would love to send other traffic, other use cases to something like it but they can't really trust its security so they don't.

Now that we have an API equivalent drop-in replacement that looks and feels even better and is entirely open source, then they can easily switch to it.

## Mission

What we want is for proliferation of these routers to exist because it's important that people have the safety, security, uptime, and cost savings from this.

I'm trying to democratize everybody in the world's access to large language models and AI.

Especially today when we're seeing increasing amounts of centralization, that's very concerning for the future and a lot of people are feeling a lot of anxiety about AI because of this.

By creating these tools and open sourcing them then we can create a way for people to have alternatives.
