Why switch

Routing convenience with a prompt path you can inspect.

TrustedRouter is for teams that want model choice, provider fallback, usage tracking, BYOK, prepaid credits, and a hosted gateway whose code path is published for verification.

The control plane does not terminate production prompt traffic. Prompts belong on api.quillrouter.com, which is the attested API path.

Developer workflow

Keep the easy parts. Make the trust boundary explicit.

Model routers are useful because they remove provider account sprawl, normalize usage records, and make fallback possible. TrustedRouter keeps that workflow and adds a published gateway boundary for production prompt traffic.
